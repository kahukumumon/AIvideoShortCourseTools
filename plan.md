# 動画編集ツール 実装計画

## 目的

- `docs/require_edit.md` の仕様に基づき、ブラウザ上で完結する動画編集ツール MVP を実装する。
- 開発ソースは配信用静的ファイルと分離し、ビルド成果物を `tools/video_edit` 配下へ出力する。
- GitHub Pages でそのまま配信できる構成を維持する。

## 完成条件

- **複数無音動画を読み込み**、各動画を **タイムラインの複数クリップ** として扱える。
  - 各動画クリップは独立のクリップ ID を持つ。
  - 動画トラック内で重複を禁止しつつ、空白区間を保持できる。
  - 動画クリップはドラッグで前後のクリップ順を入れ替えでき、たとえば 2 番目のクリップを後方へ移動すると 3 番目になる。
- 複数の音声トラックを追加でき、各トラック内でクリップ重複を禁止できる。
- 複数のモザイクトラックを追加でき、各トラック内でクリップ重複を禁止できる。
- 動画クリップと音声クリップに対して、移動、トリミング、速度変更、分割、削除、結合、フェード設定ができる。
- モザイククリップに対して、時間配置、削除、分割、結合ができる。
- モザイククリップは動画クリップと統一感のある見た目で表示される。
- モザイククリップに対して、時間配置、長さ調整（トリミング）、削除、分割、結合ができる。
- 音声クリップに対して個別音量調整と、同一音声トラック内クリップへの一括音量適用ができる。
- プレビュー上でモザイククリップの楕円形モザイクを表示し、位置、サイズ、回転を直接操作できる。
- プレビュー上で有効なモザイク枠を常時ライトグリーンで表示し、選択中クリップの枠はオレンジで強調表示できる。
- モザイク方式はピクセルモザイクで、ピクセルサイズは動画長辺の `1/100` で一貫して適用される。
- **音声タイムライン各クリップに 10ms ごとの RMS グラフを表示** し、音量ピークを視覚化できる。
  - RMS 計算は Web Audio API の AnalyserNode を使用。
  - グラフは Canvas または簡易 SVG で、クリップの背景に重ねて表示。
- **シークバーはタイムラインと共用**（動画と音声で統一）。
  - 1 本の seekbar で全クリップを制御。
  - 再生ヘッドと連動。
- プレビュー上で動画と音声を同期再生でき、再生、停止、シークが機能する。
- プレビュー上で再生ヘッド時刻に有効なモザイクを重ねて表示できる。
- 編集結果を `mp4` として書き出せる。
- 出力は `H.264 + AAC` を前提とし、複数音声トラックをミックスできる。
- 出力 `mp4` は複数動画クリップを時系列で連結したシーケンスを保持する。
- 出力 `mp4` はモザイククリップの時間範囲と楕円形状を反映する。
- 静的ビルド成果物が `tools/video_edit` に生成され、トップページから遷移できる。

## 実装方針

- 新規フロントエンドを `Vite + React` で構築する。
- 開発ソースは `src/video-edit-app` に配置する。
- 配信用ビルドは `tools/video_edit` に出力する。
- 既存サイトは静的配信のまま維持し、トップのツール一覧へ動画編集ツール導線を追加する。
- 主要な編集状態は React state と純関数ベースの timeline モデルで管理する。
- 再生処理は HTMLMediaElement と Web Audio API を併用し、プレビュー同期を取る。
- モザイクオーバーレイはプレビュー座標系と動画出力座標系を分離して管理し、比率変換で同期する。
- モザイクのピクセルサイズは動画長辺基準の正規化値として管理し、プレビューと出力で同じ見え方に寄せる。
- エクスポート処理はブラウザ内完結を維持するため `ffmpeg.wasm` を採用する。
- ピッチ維持付き速度変更は可能な限り Web Audio API と ffmpeg フィルタで揃える。

## フェーズ別タスク

### 0. 追加指摘反映（2026-04-12）

- モザイククリップの視覚スタイルを動画クリップと統一する（余白、ラベル、選択時の強調ルールを共通化）。
- プレビュー上のモザイク枠を常時ライトグリーンで表示し、編集対象クリップ選択時は当該枠のみオレンジに切り替える。
- モザイククリップに動画クリップと同等の長さ操作 UI（端ドラッグによるトリミング）を実装する。
- モザイク行の操作は行名の右に「追加」「モザイクN削除」を配置し、タイムラインクリック時の自動追加を禁止する。

### 1. 基盤構築

- `Vite + React` の開発基盤を追加する。
- Pages 配信を考慮した base path と build 出力先を設定する。
- 開発用コマンドと本番 build コマンドを README に追記する。
- トップページに `tools/video_edit` への導線を追加する。

### 2. ドメインモデル実装

- 動画クリップ、音声クリップ、音声トラック、再生ヘッドの型を定義する。
- モザイククリップ、モザイクトラック、楕円マスクの型を定義する。
- クリップ長、表示長、速度反映後長、フェード上限、音量範囲を検証するユーティリティを実装する。
- 分割、結合、移動、トリミング、速度変更、音量一括適用、モザイク変形更新の純関数を用意する。
- 同一音声トラック内の重複禁止ロジックを実装する。
- **動画トラック内の重複禁止ロジック**を実装する。
  - 動画トラックは空白区間を許可する。
  - 移動対象クリップ自身を除いた空き区間を探索し、順序入れ替えを伴う再配置を許可する。
- **モザイクトラック内の重複禁止ロジック**を実装する。
  - モザイクトラックは空白区間を許可する。
  - 別モザイクトラック間では時間重複を許可する。
- **RMS グラフ計算**: 各音声クリップについて、SourceNode 再生時に 10ms ごとの RMS 値を計算し配列化する。
  - RMS = sqrt(平均二乗) を正規化して -80dB ～ 0dB 範囲へマッピング。
  - 計算はクリップ読み込み時およびキャッシュとして保持。
- モザイクの楕円中心座標、半径、回転角度を正規化座標で保持し、プレビューとエクスポートの双方で再利用できるようにする。
- モザイクのピクセルサイズを動画長辺比で算出するユーティリティを実装する。

### 3. 編集 UI 実装

- **動画ドロップ領域は複数ファイル対応**（複数回の追加 or 複数ファイル一度に）。
  - アップロードされた動画ファイルは順次クリップ化。
  - 最初の動画の第1フレームをプレビューに表示。
- 音声ドロップ領域、プレビュー、タイムライン、再生ヘッド、シークバー、再生 UI を実装する。
- **動画トラック複数本対応**（複数動画クリップを並べる）。
- 音声トラック複数本のレイアウトを実装する。
- モザイクトラック複数本のレイアウトと追加 UI を実装する。
- モザイク行ヘッダに「追加」「モザイクN削除」操作を配置し、動画・音声行と同じ操作導線に揃える。
- ドラッグ移動、`Shift + ドラッグ` トリミング、端ドラッグによる速度変更を実装する。
  - クリップ移動は同一トラック内の空白区間へ配置可能にし、隣接クリップをまたいだ並び替えに対応する。
  - 動画トラックも空白区間を視覚的に維持したまま配置できるようにする。
- モザイククリップのドラッグ移動、分割、結合、削除をタイムライン上で実装する。
- モザイククリップにも端ドラッグによる長さ調整（トリミング）を実装し、動画クリップと同じ操作感に揃える。
- タイムラインの空き領域クリックではモザイククリップを追加しない仕様に統一する。
- 右クリックメニューから分割、結合、削除、フェード設定、音量設定を提供する。
- 音声トラック追加 UI と、トラック内一括音量適用 UI を実装する。
- **音声タイムラインの各クリップに RMS グラフをオーバーレイ**する。
  - Canvas または SVG で 10ms ごとの RMS を棒グラフ化。
  - クリップ背景に半透明で重ねて、音量ピークを視覚化。
- プレビュー上にモザイク楕円オーバーレイを描画する。
- プレビュー上のモザイク枠は非選択時ライトグリーン、選択時オレンジの色分けで常時表示する。
  - 中心移動、サイズ変更、回転ハンドル操作に対応する。
  - 選択中クリップのみ編集ハンドルを表示し、非選択時も結果プレビューは維持する。
  - ピクセルモザイクの粒度は動画長辺の `1/100` を基準に描画する。

### 4. 再生同期実装

- 再生ヘッド位置に応じて動画と全音声クリップの再生位置を同期させる。
- 再生ヘッド位置に応じて有効なモザイククリップを解決し、プレビューへ重畳する。
- 停止、シーク、再生再開で状態が破綻しないよう制御する。
- 動画速度変更時のプレビュー再生と、音声速度変更時のピッチ維持再生を実装する。
- フェードイン、フェードアウト、音量をプレビュー再生へ反映する。
- モザイクの位置、サイズ、回転の編集結果がシークや再生中断後も安定して再現されるようにする。

### 5. エクスポート実装

- タイムライン状態から ffmpeg 入力へ変換するパイプラインを構築する。
- **複数動画クリップを時系列で連結**し、映像ストリームを生成する。
  - 各動画クリップは独立に trim・速度変更を適用。
  - concat フィルタで順序通りに結合。
- 全音声トラックをフェード、速度、音量込みで加工し、1 本へミックスする。
- モザイククリップを ffmpeg フィルタへ変換し、該当時間帯に楕円形モザイクを合成する。
- ピクセルモザイクの粒度を出力動画長辺の `1/100` で計算し、ffmpeg フィルタへ反映する。
- 複数モザイクトラックに同時刻のクリップがある場合は、全てのモザイクを順に合成する。
- 複数動画ファイル間では、最初の動画の FPS と前の動画の解像度を基準に統一。
- `mp4` ダウンロードを提供する。

### 6. 検証と仕上げ

- 代表ケースの手動検証手順を整備する。
- 主要な純関数へユニットテストを追加する。
- モザイクトラック配置、楕円パラメータ更新、エクスポート用フィルタ生成のテストを追加する。
- モザイクのピクセルサイズ算出が動画長辺の `1/100` になることを確認するテストを追加する。
- ビルド成果物を `tools/video_edit` に反映する。
- README と必要ドキュメントを更新する。

## 受入観点

- 動画未投入時はエクスポートできない。
- **複数動画ファイルを追加できる**（一度に複数、または逐次追加）。
- 動画トラックは各クリップが **独立の timelineStart を持ち**、重複せず、空白区間を保持できる。
- クリップをドラッグすると、同一トラック内の空白が十分ある位置へ移動でき、前後のクリップ順を入れ替えられる。
- **シークバーは統一**（動画と音声で共有タイムラインを使用）。
- 同一音声トラック内で重なる配置を拒否または補正できる。
- 同一モザイクトラック内で重なる配置を拒否または補正できる。
- 分割位置は常に再生ヘッド基準である。
- 結合は同一トラック内の隣接クリップのみ許可される。
- 結合後は個別編集履歴を持たず、仕様どおりに属性統合される。
- フェード時間がクリップ長の `50%` を超えない。
- 音量が `-60dB` 未満、`+12dB` 超にならない。
- 再生速度が `0.25x` 未満、`4x` 超にならない。
- **音声クリップタイムラインに RMS グラフが表示される**（10ms ごとにサンプリング）。
  - RMS グラフの高さは -80dB ～ 0dB を視覚化（または正規化）。
  - クリップの背景に重ねて、音量ピークを概観できる。
- モザイクトラックを追加でき、モザイククリップを複数配置できる。
- モザイク行ヘッダに「追加」「モザイクN削除」が表示され、タイムラインクリックでクリップが勝手に追加されない。
- モザイククリップの見た目が動画クリップと同系統で、選択状態を判別できる。
- モザイククリップの長さを端ドラッグで調整できる。
- モザイククリップを選択すると、プレビュー上で楕円の位置、サイズ、回転をドラッグ操作できる。
- プレビュー上でモザイク枠は常時表示され、非選択枠はライトグリーン、選択枠はオレンジで表示される。
- モザイクのピクセルサイズが動画長辺の `1/100` としてプレビューとエクスポートで一貫する。
- プレビューで見えているモザイク形状と、エクスポート結果のモザイク形状が大きく乖離しない。
- 再生とエクスポート結果が大きく乖離しない。
- 複数動画を順に並べたエクスポート結果が、想定どおりに連結される。

## 想定リスクと対策

- ブラウザ内 `ffmpeg.wasm` は重いため、初期ロードと進捗表示を設ける。
- 長尺素材ではメモリ負荷が高いため、MVP では単一動画前提を明示し、処理中のガードを入れる。
- ブラウザだけで精密な動画編集 UI を作るため、timeline 操作ロジックは UI から分離してテスト可能にする。
- ピッチ維持付き速度変更はブラウザ差異が出やすいため、プレビューとエクスポートの両方を実機検証する。
- 楕円モザイクのプレビュー表現と ffmpeg 側フィルタ表現に差異が出やすいため、座標変換と見た目差分を重点検証する。
